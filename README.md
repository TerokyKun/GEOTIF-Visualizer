# Визуализатор алгоритма

Это веб-приложение для работы с рельефной картой в формате **GeoTIFF** и профилем высот в **TXT**.  
Проект решает задачу сопоставления эталонного профиля с рельефом карты, подбирает лучшее направление по кругу **360°**, а затем показывает результат на удобном мобильном интерфейсе.

## Что умеет проект

- работает в **демо-режиме** с предустановленными файлами;
- работает в режиме **своих файлов**;
- принимает для работы только:
  - `terrain.tif` или любой другой `*.tif / *.tiff`;
  - `heights.txt` или любой другой `*.txt`;
- `config.json` используется только для демо-набора, в режиме своих файлов он не обязателен;
- не запускает анализ, пока не выбраны нужные файлы;
- показывает:
  - карту;
  - точку старта;
  - точку назначения;
  - **профили**: референсный и лучший найденный;
  - **круг корреляций 360°**;
  - лучшее направление и все значения по окружности;
- защищает карту от случайного чрезмерного зума и от неудобного изменения масштаба на телефонах;
- использует переключатели в стиле мобильного интерфейса.

## Как работает приложение

### 1. Загрузка данных
Приложение может работать в двух режимах:

**Демо**
- автоматически подхватывает готовые данные из папки `data/`;
- сразу показывает карту и пример работы алгоритма.

**Свои файлы**
- пользователь загружает собственные `GeoTIFF` и `TXT`;
- после нажатия кнопки «Применить файлы» приложение проверяет данные и строит рабочую среду;
- если `TXT` содержит нечисловые или некорректные данные, показывается всплывающее сообщение:
  **«TXT файл содержит некорректные данные. Загрузите другой файл.»**

### 2. Работа с картой
Карта отображается как интерактивное полотно:
- можно выбирать точку 2;
- можно двигать карту;
- можно масштабировать карту;
- масштаб ограничен, чтобы нельзя было случайно сделать карту слишком маленькой;
- логика перерисовки учитывает изменение размера окна, мобильный режим и DevTools, чтобы карта не пропадала при перестройке интерфейса.

### 3. Запуск анализа
После выбора точки 2 пользователь запускает анализ.

Сервер:
- читает высоты из GeoTIFF;
- читает эталонный профиль из TXT;
- вычисляет стартовую точку в центре карты;
- вычисляет направление к точке 2;
- запускает поиск по углам **0–359°**;
- сравнивает профиль карты с референсным профилем;
- выбирает угол с максимальной корреляцией.

### 4. Отображение результата
После анализа появляются:
- лучший угол;
- коэффициент корреляции;
- список лучших направлений;
- референсный профиль;
- лучший найденный профиль;
- круговая схема корреляций 360°.

## Суть алгоритма

Алгоритм построен вокруг идеи **сравнения профиля высот по разным направлениям**.

1. Из точки старта строится луч в заданном направлении.
2. По этому лучу с карты берётся профиль рельефа.
3. Полученный профиль приводится к общей длине и шагу.
4. Он сравнивается с эталонным профилем из `TXT`.
5. Для каждого угла считается корреляция.
6. Угол с наибольшей корреляцией считается лучшим направлением.

### Что именно сравнивается
Сравниваются не просто точки, а **форма профиля**:
- подъёмы;
- спуски;
- перепады;
- общая структура рельефа.

То есть алгоритм ищет направление, в котором рельеф карты максимально похож на эталонный профиль.

## Почему используется круг 360°
Круговая схема нужна, чтобы:
- видеть весь поиск целиком;
- быстро находить максимум;
- сравнивать соседние направления;
- визуально понимать, куда указывает лучший результат.

На круге:
- каждый градус — это одно направление;
- значение корреляции показывается по окружности;
- лучший угол выделяется отдельно;
- маркер стоит на границе окружности.

## Работа с файлами

### Для демо
Используются:
- `data/terrain.tif`
- `data/heights.txt`
- `data/config.json`

### Для своих файлов
Нужны только:
- любой `*.tif` или `*.tiff`
- любой `*.txt`

Файл `config.json` в пользовательской загрузке не обязателен.

## Интерфейс

Проект сделан в стиле мобильного интерфейса:
- переключатели выглядят как ползунки;
- режимы подписаны явно;
- карта имеет отдельный ползунок для удобства;
- элементы анализа появляются только тогда, когда они действительно нужны.

## Запуск

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

После запуска открыть:

```text
http://127.0.0.1:8000
```

## Структура проекта

- `app/main.py` — FastAPI-сервер и API;
- `app/core/io.py` — чтение GeoTIFF и TXT;
- `app/core/matching.py` — поиск лучшего направления;
- `app/core/profiles.py` — построение и интерполяция профилей;
- `static/index.html` — интерфейс;
- `static/app.js` — логика фронтенда;
- `static/styles.css` — оформление.

## Кратко

Это визуализатор, который:
- берёт карту высот;
- берёт эталонный профиль;
- сравнивает их по всем направлениям;
- находит лучший угол;
- показывает результат красиво и удобно, особенно на телефоне.
